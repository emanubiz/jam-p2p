# P0.5 — Secure Network Path Validation

Manual procedure to validate WSS + room auth + dynamic TURN **end-to-end**
after unit/integration tests pass. Complements `server.room-auth.integration.test.js`
and `server.turn.integration.test.js`.

---

## Automated prerequisites (CI / agent)

| Check | Command | Expected |
|---|---|---|
| Room auth integration | `cd jam-signaler && npm test` | 67+ Jest, includes 4 room-auth cases |
| TURN REST integration | same | `/ice-servers` + Welcome with ephemeral creds |
| Rust client WSS | `cargo test --bins` in `jam-gui/src-tauri` | 36/36 |

Optional automated smoke (Docker Desktop must be running):


## Stack A — Secure dev (Windows-friendly)

Uses `docker-compose.secure-dev.yml` (coturn with port mapping, Caddy `tls internal`).

```powershell
cd jam-signaler
cp .env.example .env
# Edit .env — set both secrets (openssl rand -hex 32):
#   ROOM_AUTH_SECRET=...
#   TURN_SECRET=...
docker compose -f docker-compose.secure-dev.yml up -d --build
```

Verify HTTP (direct signaler, dev only):

```powershell
Invoke-RestMethod http://localhost:8080/health
Invoke-RestMethod http://localhost:8080/room/e2e-secure/token
# → { token: { exp, sig } } when ROOM_AUTH_SECRET is set
```

Verify WSS (Caddy → signaler):

```powershell
# PowerShell 7+ or curl.exe; accept self-signed cert once
curl.exe -k https://localhost/health
curl.exe -k https://localhost/room/e2e-secure/token
```

Tauri client:

1. Server URL: `wss://localhost`
2. Room: `e2e-secure`
3. The app fetches `GET /room/e2e-secure/token` automatically when auth is enabled.

## Stack B — Production (Linux host)

Uses `docker-compose.prod.yml` with coturn `network_mode: host` (not supported on
Docker Desktop Windows).

```bash
cd jam-signaler
cp .env.example .env
# ROOM_AUTH_SECRET, TURN_SECRET, SIGNALING_DOMAIN in Caddyfile
docker compose -f docker-compose.prod.yml up -d
```

Point clients at `wss://your.domain`.

## Validation checklist (2 clients)

| Step | Pass criteria |
|---|---|
| 1. Token mint | `GET /room/:name/token` returns `{ exp, sig }` |
| 2. Join without token | WebSocket `Error` (room auth enabled) |
| 3. Join with token | Both peers receive `PeerList` / `NewPeer` |
| 4. WSS | Client connects to `wss://` (no cleartext SDP on wire) |
| 5. TURN relay | Block host candidates (firewall) or use symmetric NAT; audio still flows |
| 6. coturn logs | `session ... new` with REST username `expiry:jam` accepted |

---

## Force TURN relay (optional)

On a Linux host with `iptables`:

```bash
# Block direct UDP between peers (example — adjust interfaces)
sudo iptables -A INPUT -p udp --dport 1024:65535 -j DROP
```

Restore after test. Confirm in coturn stdout that allocations succeed.

---

## Known limitations

- **Windows + Docker Desktop:** use `docker-compose.secure-dev.yml`; production
  coturn host networking requires Linux/macOS bare metal or VM.
- **Self-signed TLS:** browsers/webviews may require trusting Caddy's internal CA
  once (`tls internal` in Caddyfile).
- **P0 audio:** this procedure validates the **network/security path** only;
  bidirectional playback still requires `E2E-AUDIO-PROCEDURE.md`.
